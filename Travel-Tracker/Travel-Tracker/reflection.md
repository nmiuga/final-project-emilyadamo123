#  Reflection

## Overall Learnings
Building my Travel Tracker app helped me start connecting a lot of the core concepts we learned in this class into one project. Before this, everything felt kind of separate, but this app made it feel more real. I worked with real JSON data from the REST Countries API and used Codable to turn that data into Swift models. It was helpful to see how that data actually moves through the app and shows up in the UI. On the UI side, working with SwiftUI stacks, Navigation​Stack, navigation​Destination, and lightweight components like the Visited/Wishlist Bar. Styling the app (consistent colors, spacing, rounded rectangles, and capsules) showed me how much polish comes from detail-oriented design choices. 

## Challenges
One major challenge of this project was working with AI tools with a free plan with limited iteration opportunities. Oftentimes I’d run into my chat limit pretty quickly after trying prompts and troubleshooting different issues, as the early parts of the project had me rely a bit on the LLM. This made it harder to iterate quickly, since I couldn’t always ask follow-up questions right away and had to pause or switch approaches mid-problem. Because of this, I had to be more intentional with how I asked questions and often spent more time planning before using AI at all. While it slowed down the process at times, it ultimately made me more self-sufficient and improved my ability to reason through problems on my own.

## Next Steps
If I had more time to expand and refine the app, some things I would add:
- Persistence: Right now, saved countries live in memory. I’d like to persist them using SwiftData or some sort of app storage so selections survive app restarts.
- Enhanced filtering and sorting: Add region-based filters and sort options (e.g., by name, population, etc) to make browsing more powerful. A sectioned list by region could also improve navigation.
- Offline-first improvements: Cache country data and flag images for faster subsequent launches and better reliability when offline.

## Reflection on Semester
Over the course of the semester, I feel like I’ve become much more comfortable reading and writing Swift code in SwiftUI. At the beginning, a lot of the syntax and structure felt overwhelming, but working through projects like this helped everything start to click. I also got better at problem-solving on my own, whether that meant debugging errors, testing different approaches, or using documentation more effectively. Overall, this class helped me feel more confident building a complete app and understanding how AI can be used as a helpful resource.
